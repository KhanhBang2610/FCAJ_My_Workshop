Tổng quan quá trình thực tập và phát triển dự án Smart Media Analytics, được chia nhỏ thành các chặng đường theo từng tuần với mục tiêu cụ thể:

- **[Tuần 1: Khởi tạo cơ bản]({{< ref "1.1-Week1.md" >}})**: Làm quen với hệ sinh thái AWS, thực hành các dịch vụ cơ sở và khảo sát định hướng dự án.
- **[Tuần 2: Lambda & VPC]({{< ref "1.2-Week2.md" >}})**: Đi sâu nghiên cứu và cấu hình hạ tầng mạng Amazon VPC, Subnets kết hợp AWS Lambda.
- **[Tuần 3: Định hướng dự án]({{< ref "1.3-Week3.md" >}})**: Hội nhập môi trường công ty, làm quen quy trình làm việc nội bộ và chốt hướng đi phát triển dự án.
- **[Tuần 4: Backend & Docker]({{< ref "1.4-Week4.md" >}})**: Khởi tạo cấu trúc Backend FastAPI, thực hành đóng gói Docker và làm quen với ECR, ECS Fargate.
- **[Tuần 5: Database & API]({{< ref "1.5-Week5.md" >}})**: Tích hợp cơ sở dữ liệu PostgreSQL, Redis và hoàn thiện thiết kế các API quản lý cốt lõi.
- **[Tuần 6: AI Pipeline & Testing]({{< ref "1.6-Week6.md" >}})**: Triển khai các dịch vụ AI bằng Docker, đồng bộ môi trường phát triển và kiểm thử toàn bộ Backend.
- **[Tuần 7: Vector Store & Media]({{< ref "1.7-Week7.md" >}})**: Hoàn thiện luồng xử lý AI bất đồng bộ (Scene, Image Caption, Transcription) và tích hợp ChromaDB.
- **[Tuần 8: Frontend Init]({{< ref "1.8-Week8.md" >}})**: Xây dựng nền tảng Frontend với React, quản lý State tập trung và kết nối API đầu tiên.
- **[Tuần 9: Bedrock & WebSocket]({{< ref "1.9-Week9.md" >}})**: Kết nối AWS Bedrock, xử lý tải lên thời gian thực qua WebSocket và tối ưu Vector Embedding.
- **[Tuần 10: Dashboard & Audio]({{< ref "1.10-Week10.md" >}})**: Xây dựng Audio Waveform, hoàn thiện Dashboard tương tác và điều chỉnh kiến trúc AWS theo feedback.
- **[Tuần 11: End-to-End & UI/UX]({{< ref "1.11-Week11.md" >}})**: Nối thông luồng pipeline AI E2E, đóng gói cho Fargate và đại tu giao diện đạt chuẩn App-like.
- **[Tuần 12: CI/CD & Deploy]({{< ref "1.12-Week12.md" >}})**: Tự động hóa CI/CD, thiết lập CloudWatch giám sát và hoàn thiện tài liệu Workshop A-Z.
